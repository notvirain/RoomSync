Viva prep summary (study this first)
1. What RoomSync is
	- A roommate expense splitter web app.
	- Users can register, log in, create groups, add members, add shared expenses, and view balances.
2. Core stack
	- Frontend: React + Vite + Axios + React Router.
	- State/auth: React Context + localStorage token persistence.
	- Backend: Node.js + Express MVC.
	- Database: MongoDB Atlas with Mongoose.
	- Auth: JWT + bcryptjs password hashing.
3. Main backend responsibilities
	- Register and login users.
	- Protect routes using JWT middleware.
	- Create groups and manage members.
	- Add expenses and calculate balances.
	- Return JSON errors with proper status codes.
4. Main frontend responsibilities
	- Auth screens for register/login.
	- Protected dashboard and group detail screens.
	- Persist login across refresh using localStorage.
	- Show loading and error states.
	- Call the backend using the production API base URL.
5. Important deployment details
	- Backend runs on Render.
	- Frontend runs on Vercel.
	- CORS must match the exact frontend domain.
	- Vercel SPA rewrites are enabled.
6. What was fixed during polishing
	- React runtime issue was fixed with Vite React plugin.
	- Raw auth errors were cleaned up.
	- Backend startup and malformed JSON handling were hardened.
	- Dist build files were removed from git tracking.

Viva talking points (short answers)
1. Why JWT?
	- It gives stateless auth and is simple to protect routes.
2. Why bcrypt?
	- Passwords are hashed before storing, so raw passwords are never saved.
3. Why MongoDB Atlas?
	- It provides cloud-hosted database access for deployment.
4. Why use Context API?
	- It keeps auth and shared app state simple without prop drilling.
5. Why use localStorage?
	- It keeps the login session after refresh for a smoother demo.
6. Why Vite?
	- It gives a fast frontend dev/build experience.
7. Why MVC on backend?
	- It keeps routes, controllers, models, and middleware separated.
8. How balances work?
	- Each expense is split equally among selected users and balances are adjusted per payer and participant.
9. How are protected routes handled?
	- JWT middleware checks the token and denies unauthorized access.
10. How does production routing work on refresh?
	- Vercel rewrite config sends SPA routes back to index.html.
